# 認證Adobe [!DNL Experience Manager Forms] 開發人員專家

瞭解如何獲得認證 [!DNL Experience Manager Forms] 開發專家。

## 最低體驗

您應該：

* 1-3年AEM Forms經驗
* IT行業至少5年的工作經驗
* 能夠領導定制表單解決方案的開發、配置、測試和部署
* 能夠配置表單、將業務要求轉換為工作流、與外部系統整合
* 對AEM Forms基金構成的理解

此外，您應熟悉以下技術和環境：

* 設計XDP模板、XFA模板
* 建立PDF forms（靜態/動態、交互和非交互）
* 架構或XML資料綁定
* 建立表單片段
* 建立最適化表單
* 複雜形式的調試
* 配置並執行表單提交
* 生成和部署適AEM應表單的實例
* 使用表單模型
* 能夠與資料服務整合
* 工作流實現
* 對表單工AEM作台(JEE)的基本瞭解，以建立長壽命 — 短壽命流程和服務。
* 使用GuideBridge API開發Javascript客戶端
* 基於sign v government CAC知道使用哪個AEM Forms環境台式機或伺服器
* 對Java/J2EE和JavaScript的基本理解，編寫邏輯並調用不同的服務。
* 預填自適應表單
* 對OSGi、Sling元件的基本理解或知識
* 使用文檔服務
* 設AEM計器、AEM工作台、工AEM作區
* 自適應表單控制台
* Java IDE工具
* 馬文
* GIT/版本控制工具

>[!NOTE]
>
>您的認證有效期為2年，從您通過認證之日起。 我們建議盡可能採用較新版本，因為它涵蓋較新的產品功能。

**考試目標和範圍**

第1節：Forms建立和管理(36%)

* 建造Forms
* 配置Forms資料模型
* 自定義AEM Forms
* 利用GuideBridge API開發和使用Javascript ClientLib

第二節：流程和工作流(34%)

* 設計和協調表單批准工作流
* 使用Document Services API開發OSGi捆綁包
* 使用監視資料夾終結點計畫不同的非同步批處理
* 從XFA/XDP設計中產生不同的輸出
* 使用Forms或匯編器服務處理PDF
* 實施JDBC服務

第三節：管理和故障排除(18%)

* 排除日誌檔案故障並區分日誌檔案
* 執行管理活動
* 應用服務包
* 為AEMForms執行Dispatcher設定
* 導航不同的控制台

第四節：API端點和整合(8%)

* 整合 Adobe Experience Cloud
* 生成和使用各種端點服務

第五節：互動式通信(4%)

* 設計、演示和實施交互通信的流程

**現在您已成功登錄，將直接訪問以下連結。**

**練習test**

我們建議在您開始之前先享受免費練習test。 您將可以不受限制地訪問所有可用的練習test，以便進行Adobe的專家級考試。

我們的實踐test都是從與現場考試相同的藍圖發展出來的。 練習test將幫助您瞭解即時考試所涵蓋的主題。

請記住，通過練習test並不保證您會通過認證考試。

轉到 [Adobe認證考試準備門戶](https://www.certmetrics.com/adobe/candidate/gmetrix_sso.aspx){target="_blank"} 現在去做一個練習test。

**參加認證考試**

Adobe使用兩家供應商來管理受監督的考試，即PSI和Examity。

* **PSI** 提供test中心和日語考試選項（如果可用）
* **[!DNL Examity]** 提供遠程線上選項

要計畫、重新安排或取消您的考試，只需按一下選擇的供應商，然後按照螢幕上提供的說明進行操作。 就這麼簡單！

Adobe憑據管理系統中的配置檔案名稱必須與政府ID完全匹配。 如果資訊不相同或不匹配，則檢查員將終止考試，並要求您除支付額外的考試費用外，還要重新安排約會。

如果在預定的考試約會前48小時之前開始，則可能取消和重新安排。 候選人必須自行修改此約會。
