# 的認證歷程 [!DNL Adobe Analytics] 開發人員專業

了解如何取得認證 [!DNL Analytics] 開發人員專業人員。

## 最低體驗

您應：

* 6-12個月的經驗 [!DNL Adobe Analytics] 實作（網頁和行動應用程式）
* 在橋接IT與業務方面，擔任支援Analytics實作團隊的參與者，以了解業務需求如何轉化為技術解決方案(SDR)，並根據分析師提供的需求而有所體驗
* 了解需要哪種變數類型、在何處及如何收集資料、設計資料模型（資料層）、設定及維護標籤管理解決方案

體驗下列工作：

* 解譯解決方案設計檔案(SDR)，並了解所有 [!DNL Adobe Analytics] 收集的變數
* 了解及使用Adobe(Experience Platform)Launch
* 了解基線的設定 [!DNL Adobe Analytics] 維度和事件頁面名稱、連結名稱、Activity Map等，但將與專家開發人員合作以進行更多客戶實作 — 變數對應仍由專家層級開發人員擁有
* 依照AA專家提供的設計，在Launch中建立和設定規則
* 運用Web主控台除錯程式或行動應用程式系統（例如charles記錄檔），測試及驗證在變數和事件引發中擷取的資料
* 了解資料層是什麼，並了解如何讀取JavaScript物件
* 共用報表、區段和要求存取的程式，但不共用如何管理使用者群組、控管通訊協定及管理SDR。
* 提供指引並設定行動/SDK解決方案
* 了解視訊追蹤（事件型）的一般架構相關知識庫檔案
* 從Analysis Workspace執行基本報表，並了解 [!DNL Adobe Analytics] Data Warehouse
* Adobe Experience Platform Launch的基本體驗(Adobe啟動)

>[!NOTE]
>
>你的認證有效期為兩年。 建議您盡可能使用較新版本，因為它涵蓋較新的產品功能。

**目標受眾：**

* 實施專家/工程師
* 開發人員
* 架構師

**考試詳細資訊：**

```
Level: Professional (0-12 months' experience)
Passing Score: 31/50
Time: 100 mins
Delivery: Online proctored (requires camera access)
Available languages: English
Cost: $125 (global) and $950 (India)
Exam ID: AD0-E213
```

**考試目標和範圍**

第1節：Adobe Experience Cloud生態系統中的分析(14%)

* 識別Adobe Experience Cloud ID的功能
* 識別Adobe啟動的用途
* 識別如何啟用/支援 [!DNL Adobe Analytics] Adobe Experience Cloud生態系統中的標籤

第二節：以解決方案設計參考為基礎的分析策略與設計(12%)

* 給定解決方案設計參考，識別必須填入哪些資料物件
* 在技術規格檔案的一部分中，識別如何實作資料擷取

第3節：Analytics實施與設定(30%)

* 確定部署步驟 [!DNL Adobe Analytics] 程式碼
* 識別Analytics追蹤所需最低部署程式碼的功能
* 識別在中設定各種元件的步驟 [!DNL Adobe Analytics] UI
* 識別 [!DNL Adobe Analytics] 報表套裝和變數設定
* 識別資料層和資料層值的特徵
* 識別用於追蹤功能（例如連結追蹤和電子商務追蹤）的App Measurement函式

第4節：標籤管理系統(18%)

* 識別是否已使用標籤稽核來符合要求
* 識別使用Experience Launch設定網站標籤的Adobe
* 在案例中，識別如何在Launch中啟用、修改和疑難排解擴充功能Adobe
* 識別如何建立規則和資料元素

第5節：行動服務與API(8%)

* 識別 [!DNL Adobe Analytics] API（資料摘要，倉儲，資料來源，報表）
* 識別處理規則和行動服務的特徵

第6節：測試、驗證和疑難排解(18%)

* 識別常見JavaScript錯誤的含義
* 識別共同的意義 [!DNL Adobe Analytics] 變數錯誤
* 在案例中，請找出應遵循的測試和除錯驗證最佳實務
* 識別如何追蹤 [!DNL Adobe Analytics] 在其生命週期內的伺服器呼叫（即從信標請求顯示到報表中）
