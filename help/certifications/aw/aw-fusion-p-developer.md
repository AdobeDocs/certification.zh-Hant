# 認證Adobe [!DNL Workfront Fusion] 開發人員專業

了解如何成為認證的Adobe Developer Professional。

## 最低體驗

您應：

* 使用 [!DNL Workfront] API和API總管，但未必精通API呼叫
* 可使用自訂API呼叫來搭配 [!DNL Workfront] 和其他API
* 可在 [!DNL Workfront] 應用程式和其他動作，如建立、讀取、更新、刪除和搜尋
* 能夠構建可行的解決方案，在不影響業務結果的情況下產生業務結果 [!DNL Fusion] 護欄
* 選擇適當觸發模組以實現業務結果的能力
* 了解如何使用流量控制（路由、迭代器/聚合器、解釋流量）
* 執行基本故障排除和錯誤處理的能力
* 有效測試和實現情境的能力
* 可管理組織團隊和使用者
* 了解REST API Oauth2通用連接器的用途和用途

>[!NOTE]
>
>你的認證有效期為兩年。 建議您盡可能使用較新版本，因為它涵蓋較新的產品功能。

**目標受眾：**

* 自動化工程師
* 自動化、流程優化和自動化管理器
* 整合顧問、顧問、合作夥伴
* 系統整合商/合作夥伴
* 業務分析員、應用業務分析員
* 系統管理員（IT、MIS、技術、知識系統、人工智慧管理員）

**考試詳細資訊：**

* 級別：專業（0-12個月經驗）
* 傳遞分數：33/51
* 時間：102分鐘
* 傳送：線上監控（需要攝像機訪問）或測試中心監控
* 可用語言：英文
* 成本：125美元（全球）/95美元（印度）
* 考試ID:AD0-E902

**考試目標和範圍**

第1節：基礎技術概念(39%)

* 給定 [!DNL Fusion] 需要使用另一個欄位格式的情境，提供可用來正確轉換資料的可能函式
* 給定 [!DNL Fusion] 需要將函式嵌套在另一個函式內的方案，請選擇正確形成的表達式
* 確定使用團隊控制連線和其他連線存取的方法 [!DNL Fusion] 功能
* 給定 [!DNL Fusion] 案例，識別系統和功能限制
* 給定 [!DNL Fusion] 需要時區操控、區分使用者、可見組織以及需要覆寫時區的情況
* 給定 [!DNL Fusion] 方案，選擇適當 [!DNL Workfront] 模組或動作
* 給定 [!DNL Fusion] 情節，選取「對應面板」運算式和/或模組，以適當將輸入資料轉換為輸出資料
* 識別正確的方式以利用 [!DNL Fusion] 「開發工具」，用於疑難排解執行中的錯誤，或判斷對協力廠商系統進行的呼叫和回應
* 給定 [!DNL Fusion] 如果某條路徑中的資料需要在另一條路徑中，則提供集獲取解決方案，以便該資料在其他路由路徑上可用
* 識別使用或產生 [!DNL Fusion] 滿足業務需求的模板
* 給定 [!DNL Fusion] 方案，確定查看執行和/或解決錯誤的操作
* 識別 [!DNL Fusion] 滿足方案需要的方案設定
* 給定 [!DNL Fusion] 方案，選擇適當的流量控制和/或確定流量控制模組的正確組合

第二節：方案設計和架構(35%)

* 提供步驟清單，選擇要剖析JSON並轉換為套件組合的正確順序
* 給定一組要求，確定正確的資料查找方法
* 識別即時和已排程觸發器之間的差異
* 識別將文檔上載到 [!DNL Workfront]
* 根據業務需求，確定跟蹤此情況的正確方法 [!DNL Fusion] 已處理記錄（非使用者）和
* 給定 [!DNL Fusion] 方案中，確定可簡化設計和/或優化維護的機會
* 識別 [!DNL Fusion] 記錄案例的最佳實務
* 在某種情況下，找出減少資料流通過方案的機會
* 在搜索模組和以下聚合器上顯示的包數示例中，說明包檢查器處理和顯示的操作數的更改
* 標識在捆綁檢查器中找到的資訊
* 說明需要封存超過60天的案例，並選取唯一可能的方式（匯入/匯出Blueprint）
* 給出錯誤403的範例：禁止，正確選擇錯誤消息的來源和原因

第3節：測試和錯誤處理(16%)

* 識別測試計畫的元素
* 識別測試案例的元素
* 給定 [!DNL Fusion] 方案，正確確定處理不可靠服務的適當指令
* 識別要在 [!DNL Fusion] 藍本
* 識別如何追蹤、讀取和解決未完成的執行
* 給定 [!DNL Fusion] 缺少所需資料的情境，請選擇處理無效資料的方法

第4節：使用API(10%)

* 如果第三方API傳回429:請求太多，請確定要防止或處理錯誤的解決方案
* 給定 [!DNL Fusion] 無法在 [!DNL Workfront] 模組，識別正確的參考和模組類型，以使用新功能
* 如果協力廠商系統沒有專用的應用程式，請識別HTTP應用程式並選取適當的模組
* 識別CRUD作業與REST API其他常見功能之間的差異
