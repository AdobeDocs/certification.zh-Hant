# 認證Adobe [!DNL Workfront Fusion] 開發人員專業人員

學習如何獲得Adobe Developer專業認證。

## 最低體驗

您應該：

* 使用 [!DNL Workfront] API和API瀏覽器，但不一定精通API調用
* 使用自定義API調用的能力 [!DNL Workfront] 和其他API
* 使用模組 [!DNL Workfront] 應用程式和其他操作，如建立、讀取、更新、刪除和搜索
* 構建可行的解決方案，在不影響業務成果的情況下實現業務成果的能力 [!DNL Fusion] 護欄
* 能夠選擇適當的觸發器模組以完成業務結果
* 瞭解如何使用流控制（路由、迭代器/聚合器、解釋流）
* 能夠執行基本故障排除和錯誤處理
* 有效test和接受情景的能力
* 管理組織團隊和用戶的能力
* 對REST API Oauth2通用連接器的用途和用途的理解

>[!NOTE]
>
>您的認證有效期為2年，從您通過認證之日起。 我們建議盡可能採用較新版本，因為它涵蓋較新的產品功能。

**目標受眾：**

* 開發人員，自動化工程師
* 自動化、流程優化和自動化管理器
* 整合顧問、顧問、合作夥伴
* 系統整合商/合作夥伴
* 業務分析員、應用業務分析員
* 系統管理員（IT、MIS、技術、知識系統、AI管理員）

**考試詳細資訊：**

* 級別：專業（0-12個月經驗）
* 通過分數：33/51
* 時間：102分鐘
* 交貨：線上監控（需要攝像頭訪問）或test中心監控
* 可用語言：英語
* 成本：125美元（全球）/95美元（印度）
* 考試ID:AD0-E902

**考試目標和範圍**

第1節：基本技術概念(39%)

* 給定 [!DNL Fusion] 使用需要使用其他欄位格式的一個欄位格式的方案，提供可用於正確轉換資料的可能函式
* 給定 [!DNL Fusion] 需要將函式嵌套在另一個函式中的方案，請選擇格式正確的表達式
* 確定使用團隊控制對連接和其他 [!DNL Fusion] 特徵
* 給定 [!DNL Fusion] 方案，確定系統和功能限制
* 給定 [!DNL Fusion] 需要時區操作、區分用戶、可見組織和需要覆蓋時區的場景
* 給定 [!DNL Fusion] 方案，選擇相應的 [!DNL Workfront] 模組或操作
* 給定 [!DNL Fusion] 方案，選擇將適當轉換輸入資料以輸出資料的映射面板表達式和/或模組
* 確定正確的方法以利用 [!DNL Fusion] 開發工具，用於排除執行中的錯誤或確定對第三方系統的呼叫和響應
* 給定 [!DNL Fusion] 如果在另一路徑中需要一條路徑中的資料，則提供集獲取解決方案，使該資料在附加路由路徑上可用
* 確定使用或生產 [!DNL Fusion] 滿足業務需求的模板
* 給定 [!DNL Fusion] 方案，確定查看執行和/或解決錯誤的方法
* 確定 [!DNL Fusion] 方案設定以滿足方案需要
* 給定 [!DNL Fusion] 方案，選擇適當的流量控制和/或確定流量控制模組的正確組合

第二節：方案設計和體系結構(35%)

* 給定步驟清單，選擇要分析JSON並轉換為捆綁的正確序列
* 給定一組要求，確定正確的資料查找方法
* 標識即時觸發器和計畫觸發器之間的差異
* 確定將文檔上載到的正確模組 [!DNL Workfront]
* 根據業務要求，確定正確的跟蹤方法 [!DNL Fusion] 已處理記錄（不是用戶）和
* 給定 [!DNL Fusion] 方案，確定簡化設計和/或優化維護的機會
* 識別 [!DNL Fusion] 記錄方案的最佳做法
* 在情況下，確定減少資料流通過情景的機會
* 給出在搜索模組和以下聚合器上顯示的綁定數示例，說明捆綁檢查器處理和顯示的操作數的變化
* 標識在捆綁檢查器中找到的資訊
* 說明需要將方案存檔超過60天並選擇唯一可能的方法（導入/導出藍圖）
* 給出了錯誤403的示例：已禁止，正確選擇錯誤消息的源和原因

第三節：測試和錯誤處理(16%)

* 確定test計畫的要素
* 標識test案例的元素
* 給定 [!DNL Fusion] 情形，正確確定處理不可靠服務的適當指令
* 標識要在中添加自定義錯誤處理的進程 [!DNL Fusion] 場景
* 確定如何跟蹤、讀取和解決未完成的執行
* 給定 [!DNL Fusion] 缺少所需資料的方案，選擇處理無效資料的方法

第四節：使用API(10%)

* 給定返回429的第三方API:請求太多，請確定防止或處理錯誤的解決方案
* 給定 [!DNL Fusion] 新功能在 [!DNL Workfront] 模組，確定正確的引用和模組類型以使用新功能
* 給定沒有專用應用的第三方系統，識別HTTP應用並選擇相應的模組
* 確定CRUD操作與REST API的其他通用功能之間的差異

**參加認證考試**

Adobe使用兩家供應商來管理受監督的考試，即PSI和Examity。

* **PSI** 提供test中心和日語考試選項（如果可用）
* **[!DNL Examity]** 提供遠程線上選項

要計畫、重新安排或取消您的考試，只需按一下選擇的供應商，然後按照螢幕上提供的說明進行操作。 就這麼簡單！

Adobe憑據管理系統中的配置檔案名稱必須與政府ID完全匹配。 如果資訊不相同或不匹配，則檢查員將終止考試，並要求您除支付額外的考試費用外，還要重新安排約會。

如果在預定的考試約會前48小時之前開始，則可能取消和重新安排。 候選人必須自行修改此約會。
