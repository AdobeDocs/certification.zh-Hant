# 認證Adobe [!DNL Commerce] 建築師大師

瞭解如何成為認證Adobe架構師主管。

## 最低體驗

您至少應具有3-5年的領導經驗 [!DNL Commerce] 開發項目，並能夠通過利用Adobe設計、整合、實施和解釋業務問題的解決方案 [!DNL Commerce] 以及相關服務。

此外，您應非常熟悉以下活動和技術：

* 擴展核心功能
* 配置Adobe [!DNL Commerce] 充分
* 建立擴展應如何工作的完整藍圖、公司對擴展應如何工作的要求，而無需進入代碼
* 瞭解業務需求/功能
* 解決方案和擴展的效能影響
* 服務合同邏輯
* GraphQL
* 將Composer理解為依賴關係工具
* 設定和配置工作實例Adobe [!DNL Commerce] 從頭開始
* 自定義管理儀表板，建立網格和表單
* 快取和索引管理；建立自定義索引
* 建立新快取類型
* 排除並調試環境中的各種問題
* 擴展和管理資料庫架構
* 安全功能，提供安全安裝和開發擴展
* 執行基於伺服器的服務 [!DNL Commerce] 連接（Redis，清漆配置管理）
* 瞭解最佳編碼標準實踐、編碼標準、PSR、SOLID原理
* 設定和使用 [!DNL Commerce] 服務連接器（和相關服務）
* 暫存內容管理
* 瞭解Adobe [!DNL Commerce] 預設值，當其可用且不可用時
* 全面掌握任何相關資訊 [!DNL Commerce] 域
* 設計項目體系結構
* 優化Adobe效能 [!DNL Commerce] 預設
* 建立和修改B2B
* AMQP協定和Adobe中的適配器 [!DNL Commerce] (S3)

**考試目標和範圍**

第1節：設計(46%)

* 設計並實施最佳Adobe解決方案 [!DNL Commerce] 滿足業務需求
* 設計邏輯和技術流程
* 自定義 [!DNL Commerce] 特徵
* 整合Adobe [!DNL Commerce] 具有外部系統和服務
* 排除設計流程故障

第2節 — 審查(32%)

* 審查和重新調整現有Adobe [!DNL Commerce] 自定義
* 利用 [!DNL Commerce] test整個工作流中的框架
* 優化效能和可擴充性以實現Adobe [!DNL Commerce]
* 排除故障以確定問題的根本原因Adobe [!DNL Commerce]
* 強制執行編碼標準

第三節：配置和部署(22%)

* 配置Adobe [!DNL Commerce] 並確保項目得到優化設定
* 配置Adobe的所有方面 [!DNL Commerce] 雲
* 監督並改進部署流程
* 解決基礎架構和配置問題

第四節：法規遵從性/安全基礎知識(13%)

* 瞭解隱私法和支付安全合規性的基本知識
* 確定Adobe的常見安全方面 [!DNL Commerce] 項目
* 確定符合可訪問性的最佳做法和法律要求
