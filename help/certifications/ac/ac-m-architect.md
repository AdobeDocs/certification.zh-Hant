# 認證Adobe [!DNL Commerce] 架構師主

了解如何成為認證的Adobe架構師主師。

## 最低體驗

您至少應有3至5年的領導經驗 [!DNL Commerce] 開發項目，並能夠通過利用Adobe來設計、整合、實施和解釋業務問題的解決方案 [!DNL Commerce] 以及相關服務。

此外，您應非常熟悉下列活動和技術：

* 擴充核心功能
* 配置Adobe [!DNL Commerce] 完全
* 建立擴充功能運作方式的完整藍圖、從公司需求到運作方式的藍圖，而無須進入程式碼
* 了解業務需求/功能
* 解決方案和擴充功能的效能影響
* 服務合同邏輯
* GraphQL
* 了解撰寫器作為相依性工具
* 設定和配置Adobe的工作實例 [!DNL Commerce] 從頭開始
* 自訂管理控制面板，建立格線和表單
* 快取和索引管理；建立自定義索引
* 建立新快取類型
* 針對環境中的各種問題進行疑難排解和除錯
* 擴展和管理資料庫架構
* 安全功能，提供安全的安裝和開發擴展
* 執行基於伺服器的服務， [!DNL Commerce] 連接（Redis，清漆配置管理）
* 了解最佳編碼標準實踐，編碼標準，PSR,SOLID原理
* 設定與使用 [!DNL Commerce] 服務連接器（及相關服務）
* 測試內容管理
* 了解Adobe [!DNL Commerce] 預設值，當其可用時，則不可用
* 全面掌握任何相關資訊 [!DNL Commerce] 網域
* 設計專案架構
* 最佳化Adobe效能 [!DNL Commerce] 預設
* 建立和修改B2B
* AMQP協定和Adobe中的適配器 [!DNL Commerce] (S3)

>[!NOTE]
>
>你的認證有效期為兩年。 建議您盡可能使用較新版本，因為它涵蓋較新的產品功能。

**考試目標和範圍**

第1節：設計(46%)

* 設計並實作最佳的Adobe解決方案 [!DNL Commerce] 滿足業務需求
* 設計邏輯和技術流程
* 自訂 [!DNL Commerce] 功能
* 整合Adobe [!DNL Commerce] 與外部系統和服務
* 疑難排解設計流程

第2款 — 審查(32%)

* 審查並重新調整現有Adobe [!DNL Commerce] 自訂
* 利用 [!DNL Commerce] 在整個工作流程中測試框架
* 優化效能和可擴充性以實現Adobe [!DNL Commerce]
* 疑難排解，找出問題的根本原因Adobe [!DNL Commerce]
* 強制編碼標準

第3節：配置和部署(22%)

* 配置Adobe [!DNL Commerce] 確保項目設定得最佳
* 配置Adobe的所有方面 [!DNL Commerce] 雲
* 監督並改進部署流程
* 疑難排解基礎架構和設定問題

第4節：法規遵從性/安全基本知識(13%)

* 了解隱私權法與支付安全性的法規遵循基礎
* 識別Adobe的常見安全性 [!DNL Commerce] 專案
* 確定無障礙合規性的最佳做法和法律要求
