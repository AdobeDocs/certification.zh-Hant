# 認證Adobe [!DNL Commerce] 開發人員專家

瞭解如何成為認證的Adobe Developer專家。

## 最低體驗

你至少應該有1-3年的Adobe [!DNL Commerce] 並熟悉以下技術和環境：

* Adobe [!DNL Commerce]  — 版本2.4
* PHP
* XDebug/PHP調試工具、瀏覽器調試工具
* 連結工具
* 作曲家
* SSH，基本Linux
* 雷迪斯，Elasticsearch, SQL，清漆，RabbitMQ
* API驗證工具、API客戶端工具(Postman)
* 版本控制軟體，如Git
* 雲架構

>[!NOTE]
>
>您的認證有效期為2年，從您通過認證之日起。 我們建議盡可能採用較新版本，因為它涵蓋較新的產品功能。

**考試目標和範圍**

第1節：Adobe [!DNL Commerce] 體系結構和定制技術(36%)

* 演示如何在Adobe中有效使用快取 [!DNL Commerce]
* 生成、使用和處理自定義擴展屬性
* 建議如何將主題自定義更新應用於產品/類別頁的解決方案
* 正確應用觀察器、首選項和插件（排序順序的影響）
* 演示在Adobe中使用配置層的能力 [!DNL Commerce]
* 演示路線在Adobe中如何工作的知識 [!DNL Commerce]
* 演示自定義頁面生成器的功能
* 確定在單個實例上配置多個站點的效果和約束
* 描述依賴項注入的功能和約束
* 描述如何在儲存設定中添加和配置欄位
* 解釋Git修補程式的使用案例和Composer中的檔案級修改
* 在CLI中建立新命令
* 演示如何編寫整合test
* 識別Adobe [!DNL Commerce] 安全功能（CSP、轉義、表單鍵、清理、reCAPTCHA、輸入驗證）
* 解釋CRON調度系統的工作原理
* 演示載入和處理資料的能力
* 演示使用App模擬的能力

第二節：使用資料庫和EAV(10%)

* 以寫程式方式處理EAV屬性和屬性集
* 演示擴展資料庫模式的能力
* 演示從Adobe Commerce導入/導出資料的能力
* 描述如何使用修補程式和循環設定來修改資料庫

第三節：使用管理員開發(6%)

* 演示更新和建立網格和表單的能力
* 擴展網格操作
* 演示建立修改量類的能力
* 演示限制訪問ACL的能力

第四節：自定義目錄(6%)

* 將更改應用於現有產品類型並建立新產品類型
* 修改和擴展現有目錄實體
* 演示管理索引和自定義價格輸出的能力
* 解釋多來源庫存如何影響庫存（方案水準）

第五節：自定義銷售操作(6%)

* 演示開發新付款方法或自定義現有付款方法的能力
* 演示添加和定制發運方法的能力
* 演示定制銷售操作的能力
* 解釋如何自定義合計

第6節：API和服務(4%)

* 演示建立新API或擴展現有API的能力
* 演示使用排隊系統的能力

第七節：Adobe [!DNL Commerce] 雲體系結構(16%)

* 演示Adobe [!DNL Commerce] 體系結構/環境工作流
* 演示有關雲項目檔案、權限和結構的工作知識
* 演示基於Adobe建立多域儲存的能力 [!DNL Commerce] 雲（多域站點將開發工作和支援混為一談）
* 演示應用程式服務的一般知識以及如何管理這些服務(YAML、PHP、MariaDB、Redis、RabbitMQ等)
* 確定如何訪問不同類型的日誌
* 演示部署項目的能力（部署的主要步驟）
* 定義ECE工具提供的功能
* 確定ECE修補程式的用途（安全違規）
* 介紹如何維護和升級ECE工具
* 區分何時聯繫支援*Yaml檔案和限制（DIY與支援票證）
* 演示OOTB REBIES功能配置和安裝的基本知識

第八節：設定/配置Adobe [!DNL Commerce] 雲(10%)

* 介紹如何設定/配置Adobe [!DNL Commerce] 雲
* 應用基本雲疑難解答知識（Web UI和變數的層次結構、配置優先順序）
* 演示對雲用戶管理和登錄UI的瞭解
* 描述如何使用UI更新雲變數
* 使用UI描述環境管理
* 演示使用UI瞭解分支
* 確定Adobe商務雲計畫功能

第九節： [!DNL Commerce] 雲CLI工具（管理部件）(6%)

* 演示使用CLI瞭解分支

**現在您已成功登錄，將直接訪問以下連結。**

**練習test**

我們建議在您開始之前先享受免費練習test。 您將可以不受限制地訪問所有可用的練習test，以便進行Adobe的專家級考試。

我們的實踐test都是從與現場考試相同的藍圖發展出來的。 練習test將幫助您瞭解即時考試所涵蓋的主題。

請記住，通過練習test並不保證您會通過認證考試。

轉到 [Adobe認證考試準備門戶](https://www.certmetrics.com/adobe/candidate/gmetrix_sso.aspx){target="_blank"} 現在去做一個練習test。

**參加認證考試**

Adobe使用兩家供應商來管理受監督的考試，即PSI和Examity。

* **PSI** 提供test中心和日語考試選項（如果可用）
* **[!DNL Examity]** 提供遠程線上選項

要計畫、重新安排或取消您的考試，只需按一下選擇的供應商，然後按照螢幕上提供的說明進行操作。 就這麼簡單！

Adobe憑據管理系統中的配置檔案名稱必須與政府ID完全匹配。 如果資訊不相同或不匹配，則檢查員將終止考試，並要求您除支付額外的考試費用外，還要重新安排約會。
