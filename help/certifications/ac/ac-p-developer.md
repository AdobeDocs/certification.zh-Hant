# 認證Adobe [!DNL Commerce] 開發人員專業

了解如何成為認證的Adobe Developer Professional。

## 最低體驗

你應該樂於在Adobe工作 [!DNL Commerce] 對Adobe有基本的認識 [!DNL Commerce] 管理開發、架構、EAV/資料庫、版面/UI、結帳和銷售，以及目錄。

此外，您也應熟悉下列技術和環境：

* Adobe [!DNL Commerce]  — 版本2.4
* 了解PHP
* MYSQL知識
* 物件導向寫程式知識
* 雲架構知識

>[!NOTE]
>
>你的認證有效期為兩年。 建議您盡可能使用較新版本，因為它涵蓋較新的產品功能。

>[!BEGINTABS]

>[!TAB 準備]

**目標受眾：**

* 開發人員
* 潛在開發人員
* 後端開發人員
* 解決方案架構師
* 技術領導者

**考試詳細資訊：**

* 級別：專業（0-12個月經驗）
* 傳遞分數：50/77
* 時間：154分鐘
* 傳送：線上監控（需要攝像頭訪問）
* 可用語言：英文
* 成本：125美元（全球）/95美元（印度）
* 考試ID:AD0-E717

**準備情況自評**

完成 [線上準備問卷](https://scorpion.caveon.com/launchpad/ad-q-e717-readiness-questionnaire-for-adobe-commerce-developer-professional/ad-q-e717-readiness-questionnaire-for-adobe-commerce-developer-professional) 來查看您目前的體驗是否符合建議的最低候選資格。

**考試目標和範圍**

第1節：與管理員合作(5%)

* 說明ACL如何與角色和資源搭配使用
* 識別建立或修改管理格線/表單時要使用的元件
* 識別建立存放區/管理員設定和功能表項目時要使用的檔案

第二節：架構(29%)

* 說明 [!DNL Magento] 檔案結構
* 說明 [!DNL Magento] CLI命令
* 說明cron功能
* 在案例中，說明di.xml的使用方式
* 在案例中，建立控制器
* 描述模組結構
* 說明索引功能
* 說明本地化
* 描述插件、首選項、事件觀察器和截獲器
* 說明自訂模組路由
* 說明URL重新寫入
* 說明 [!DNL Magento] 快取系統
* 說明商店、網站和商店檢視（基本了解）

第3節：EAV/資料庫(13%)

* 在案例中，變更/新增/移除屬性集和/或屬性
* 說明不同類型的屬性
* 在方案中，使用資料庫架構更改資料庫表
* 描述模型、資源模型和集合
* 說明實體屬性值(EAV)的基本知識

第4節：版面/UI(15%)

* 將變更套用至現有的產品類型並建立新的產品類型
* 修改和擴展現有目錄實體
* 演示管理索引和自定義價格輸出的能力
* 說明多來源庫存如何影響庫存（方案層）

第5節：結帳和銷售(8%)

* 說明購物車元件
* 說明購物車促銷規則
* 在案例中，說明基本的結帳修改
* 在案例中，說明報價資料的基本用法
* 在方案中，配置付款和發運方法
* 在案例中，設定稅則、貨幣、購物車和/或結帳

第6節：目錄(8%)

* 確定類別管理和產品管理的基本知識
* 說明產品類型
* 描述價格規則
* 說明價格類型

第7節：Adobe [!DNL Commerce] 雲架構(12%)

* 定義Adobe [!DNL Commerce] 架構/環境工作流程
* 說明雲端專案檔案、權限和結構
* 列出可用的Adobe [!DNL Commerce] 在雲端
* 說明如何存取不同類型的記錄檔
* 說明應用修補程式的步驟（確定要放置修補程式的資料夾）
* 說明如何維護和升級ECE工具
* 識別何時需要呼叫支援*Yaml檔案和限制（DIY與支援票證）

第8節：設定/設定Adobe [!DNL Commerce] 雲(4%)

* 識別如何設定/設定Adobe [!DNL Commerce] 雲
* 定義基本雲端疑難排解（Web UI和變數的階層，設定優先順序）
* 識別雲端使用者管理和入門UI的基本知識
* 使用UI說明環境管理
* 使用UI說明分支
* 識別Adobe商務雲計畫功能

第9節： [!DNL Commerce] Cloud CLI工具（管理部件）(6%)

* 說明Adobe的獨特功能 [!DNL Commerce] 雲CLI工具(CLI獨有功能：激活電子郵件，資料庫環境，快照，資料庫轉儲，本地環境設定)
* 使用Adobe說明分支 [!DNL Commerce] 雲CLI工具
* 確定連接雲服務的方式？ （我的SQL, Redis, tunnel:info）
